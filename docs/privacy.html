Privacy Policy

Effective Date: November 27, 2025

AxelBase – SHA-512 Hash Generator is a completely client-side web application. This means we collect absolutely no personal data, usage statistics, or file contents. This Privacy Policy is intentionally short because there is almost nothing to disclose.

1. What We Do NOT Collect

  • No personal information — names, emails, IP addresses, or device identifiers
  • No file contents — all hashing occurs locally in your browser using the Web Crypto API
  • No analytics or tracking — no Google Analytics, no cookies, no pixels, no telemetry
  • No server logs — the site is statically hosted on GitHub Pages with zero backend

2. How Your Data Is Handled

When you use AxelBase:

  • Text input and file processing happen entirely in your browser’s memory
  • Large files are streamed in chunks — never fully loaded into RAM
  • Copy-to-clipboard and export features use only native browser functions
  • All operations are stateless and ephemeral

Your data never touches any server — not ours, not a CDN, not a third party.

3. Third-Party Services

We use only two CDNs for performance:

  • Bootstrap 5 CSS/JS (jsDelivr)
  • Bootstrap Icons (jsDelivr)

These load static files only. They do not set cookies or track users when used on AxelBase.

4. Local Storage & PWA

When installed as a Progressive Web App, AxelBase may cache assets locally for offline use. No user-generated content is ever persisted unless you explicitly use browser features (e.g., bookmarks).

5. Security & Responsibility

While we cannot access your data, you remain responsible for:

  • Using the tool on a trusted device
  • Not sharing sensitive hashes publicly
  • Keeping your browser updated

6. Changes to This Policy

Any updates will appear on this page with a revised effective date.

Your privacy is not a feature — it’s the entire architecture.
By using AxelBase, you agree to this zero-data-collection policy.
